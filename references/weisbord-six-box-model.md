# 组织诊断访谈指南 - 韦斯伯德六个盒子模型

本指南为使用韦斯伯德六个盒子模型进行组织诊断访谈提供详细指导。

---

## 📊 核心模型介绍

该模型由 Marvin Weisbord 于 1976 年提出，用于从六个维度系统评估组织健康度。

### 六个盒子

1. **目的（Purpose）**: 目标是否清晰、被理解、被认同？
2. **结构（Structure）**: 组织架构、分工、流程是否支持目标实现？
3. **关系（Relationships）**: 人际关系、团队协作、冲突处理机制如何？
4. **奖励（Rewards）**: 激励机制是否公平、有效、与目标对齐？
5. **领导（Leadership）**: 领导力、决策、方向指引是否明确？
6. **支持机制（Helpful Mechanisms）**: 工具、流程、信息、培训等支持是否到位？

---

## 🎯 六个维度详细诊断框架

### 1. 目的（Purpose）

**定义**: 组织的目标、使命、愿景是否清晰并被认同

**诊断问题**:
- "公司的战略目标清晰吗？"
- "你理解团队在公司战略中的角色吗？"
- "你的工作目标与公司目标对齐吗？"

**常见问题信号**:
- 目标频繁变动
- 员工不理解目标
- 目标与实际工作脱节

**如何区分系统性问题与个人抱怨**:
- **系统性**: 多人反映同样问题，跨团队存在
- **个人**: 仅个别员工反映，与其角色相关

### 2. 结构（Structure）

**定义**: 组织架构、职责分工、流程设计

**诊断问题**:
- "组织架构清晰吗？职责分工明确吗？"
- "流程顺畅吗？有卡点吗？"
- "授权机制合理吗？"

**常见问题信号**:
- 职责不清，推诿责任
- 流程冗长，效率低下
- 授权不足或过度授权

**区分方法**:
- **系统性**: 跨部门流程问题，多团队类似问题
- **个人**: 对特定流程不熟悉

### 3. 关系（Relationships）

**定义**: 人际关系、团队协作、冲突处理

**诊断问题**:
- "团队氛围怎么样？"
- "跨团队协作顺畅吗？"
- "冲突如何处理？"

**常见问题信号**:
- 团队内部冲突
- 跨部门协作困难
- 沟通不畅

**区分方法**:
- **系统性**: 多团队反映协作问题，文化问题
- **个人**: 特定人际冲突

### 4. 奖励（Rewards）

**定义**: 激励机制的公平性、有效性、与目标对齐度

**诊断问题**:
- "激励机制公平吗？"
- "激励与绩效匹配吗？"
- "奖励及时吗？"

**常见问题信号**:
- 绩效与奖励不匹配
- 激励不透明
- 奖励不及时

**区分方法**:
- **系统性**: 多人反映，跨部门存在
- **个人**: 对自身奖励不满意

### 5. 领导（Leadership）

**定义**: 领导力、决策机制、方向指引

**诊断问题**:
- "领导力有效吗？"
- "决策流程清晰吗？"
- "方向指引明确吗？"

**常见问题信号**:
- 决策缓慢或混乱
- 领导力不足
- 方向不清晰

**区分方法**:
- **系统性**: 多团队反映，组织层面问题
- **个人**: 对特定领导不满

### 6. 支持机制（Helpful Mechanisms）

**定义**: 工具、流程、信息、培训等支持

**诊断问题**:
- "工具和资源充足吗？"
- "信息获取顺畅吗？"
- "培训体系完善吗？"

**常见问题信号**:
- 工具落后
- 信息不对称
- 培训不足

**区分方法**:
- **系统性**: 多团队反映，跨部门问题
- **个人**: 特定技能或资源需求

## 🔧 维度选择算法

根据员工角色自动选择最相关的维度：

```python
def select_dimensions(employee_role):
    """
    根据员工角色选择最相关的维度（3-6 个）
    """
    dimension_priority = {
        '高管': ['目的', '领导', '结构', '奖励'],
        '中层管理': ['目的', '结构', '关系', '领导'],
        '基层管理': ['结构', '关系', '支持机制', '奖励'],
        '一线员工': ['支持机制', '关系', '奖励'],
        '技术员工': ['支持机制', '结构', '奖励'],
        '新员工': ['支持机制', '关系', '目的'],
        '老员工': ['目的', '奖励', '支持机制']
    }
    return dimension_priority.get(employee_role, ['支持机制', '关系', '奖励'])
```

### 示例

- **高管**: 重点问目的、领导、结构
- **中层管理**: 重点问目的、结构、关系
- **一线员工**: 重点问支持机制、关系、奖励
- **技术员工**: 重点问支持机制、结构、奖励
- **新员工**: 重点问支持机制、关系、目的
- **老员工**: 重点问目的、奖励、支持机制

## 💬 追问技巧：引导系统性思考

### 核心原则

1. **从具体到抽象**: 先问具体问题，再上升到机制层面
2. **从个人到组织**: 先问个人体验，再问组织层面
3. **从现象到本质**: 先问表现，再问根源
4. **从系统到建议**: 先问问题，再问改进建议

### 追问模板

```
现象层 → 影响层 → 机制层 → 系统层 → 建议层

示例：
现象：你遇到过什么协作困难？
影响：这些困难对你的工作有什么影响？
机制：这些困难的根源是什么？
系统：这是系统性问题还是个别情况？
建议：你认为应该如何改进？
```

### 常见陷阱

| 陷阱 | 表现 | 避免 |
|-----|------|------|
| 引导性提问 | 暗示答案 | 保持中立 |
| 过早下结论 | 未充分了解就判断 | 多听少说 |
| 混淆层面 | 个人问题与系统问题混淆 | 明确区分 |
| 忽视数据 | 仅凭印象判断 | 基于事实和数据 |

## 📊 访谈后分析

### 问题分类矩阵

| 影响范围 | 发生频率 | 问题类型 | 处理方式 |
|---------|---------|---------|---------|
| 个人 | 偶发 | 个人问题 | 个人辅导、沟通 |
| 团队 | 频发 | 流程优化 | 团队内部改进 |
| 跨团队 | 频发 | 团队协调 | 跨团队协作机制 |
| 全组织 | 频发 | 系统变革 | 组织级改进项目 |

### 优先级处理

| 优先级 | 问题类型 | 处理时间 |
|-------|---------|---------|
| 🔴 高 | 影响大、频发的系统性问题 | 立即启动改进项目 |
| 🟡 中 | 影响中、偶发的流程问题 | 纳入改进计划 |
| 🟢 低 | 影响小、偶发的个人问题 | 个人辅导、沟通 |

## ⚠️ 注意事项

### 访谈伦理
- **保密原则**: 严格保密访谈内容
- **中立立场**: 不预设结论，客观记录
- **知情同意**: 告知目的和保密安排
- **心理安全**: 营造安全环境，鼓励坦诚反馈

### 常见偏差

| 偏差类型 | 表现 | 避免 |
|---------|------|------|
| **近因效应** | 过于关注最近发生的事 | 回顾更长时间跨度 |
| **从众效应** | 受他人意见影响 | 独立访谈，避免引导 |
| **社会期许** | 回答符合社会期望 | 强调匿名和安全 |
| **负面偏好** | 过度关注负面信息 | 平衡正负面信息 |
