# 杨三角理论 - 组织能力诊断框架

## 📖 理论概述

**杨三角理论**（Yang's Triangle）由腾讯高级管理顾问杨国安教授提出，是评估和组织能力建设的经典框架。

### 核心公式

```
企业持续成功 = 战略 × 组织能力
```

- **战略**：方向选择（做什么、不做什么）
- **组织能力**：执行能力（能否做到）
- **关系**：乘法而非加法，任一为零则整体为零

### 杨三角模型

```
                    员工思维
                   (Mindset)
                      ▲
                     / \
                    /   \
                   /     \
                  /       \
                 /         \
                /           \
               /             \
              /               \
             /                 \
            /                   \
           /                     \
          /                       \
         /                         \
        /                           \
       /                             \
      /                               \
     /                                 \
    /                                   \
   /                                     \
  /                                       \
 /                                         \
员工能力 ◄─────────────────────────► 员工治理
(Employee                                   (Governance)
 Competence)
```

**三个支柱：**

| 支柱 | 核心问题 | 关键要素 |
|-----|---------|---------|
| **员工思维** | 员工是否愿意？ | 价值观、使命感、工作意愿、激励机制 |
| **员工能力** | 员工是否会？ | 知识、技能、人才梯队、培养体系 |
| **员工治理** | 员工是否被允许？ | 组织架构、流程制度、授权机制、信息平台 |

---

## 🔍 三大支柱详解

### 一、员工思维（Mindset）- "愿不愿意"

**诊断重点：** 员工是否有正确的价值观、工作动机和投入度

#### 关键诊断维度

```
1. 价值观对齐
   - 员工是否认同公司/团队的使命和价值观？
   - 个人目标与组织目标是否一致？
   - 工作中是否有意义感和成就感？

2. 工作动机
   - 员工是被动执行还是主动投入？
   - 内在动机（兴趣、成长）vs 外在动机（薪酬、晋升）
   - 工作热情和敬业度如何？

3. 激励机制
   - 现有激励是否能驱动期望行为？
   - 物质激励与精神激励的平衡
   - 短期激励与长期激励的组合

4. 文化氛围
   - 团队氛围是积极向上还是消极抱怨？
   - 是否鼓励创新和试错？
   - 员工是否有归属感和安全感？
```

#### 典型问题示例

```
【价值观对齐】
- 你觉得我们团队存在的意义是什么？
- 你个人的工作目标是什么？和团队目标有什么关系？
- 工作中什么事情让你觉得最有意义？

【工作动机】
- 你每天早上起床愿意来工作的原因是什么？
- 如果不用考虑钱，你还会选择现在的工作吗？为什么？
- 什么事情会让你在工作中感到兴奋和有动力？

【激励机制】
- 你觉得什么样的奖励最能激励你？
- 过去一年中，哪次认可让你印象最深刻？
- 你觉得付出和回报匹配吗？

【文化氛围】
- 用三个词形容我们团队的氛围，会是什么？
- 团队中提出新想法通常会发生什么？
- 当你犯错时，团队的反应是什么？
```

#### 常见问题信号

| 信号 | 可能问题 | 追问方向 |
|-----|---------|---------|
| "就是份工作而已" | 缺乏意义感 | 探索个人价值观与工作连接 |
| "领导让做什么就做什么" | 被动执行 | 了解授权和自主性 |
| "干好干坏差不多" | 激励失效 | 诊断激励公平性和有效性 |
| "多一事不如少一事" | 心理安全缺失 | 了解试错文化和容错机制 |

---

### 二、员工能力（Employee Competence）- "会不会"

**诊断重点：** 员工是否具备完成工作所需的知识、技能和能力

#### 关键诊断维度

```
1. 能力匹配度
   - 现有能力是否匹配岗位要求？
   - 能力缺口在哪里？
   - 是否有清晰的能力标准？

2. 人才培养
   - 是否有系统的培训体系？
   - 学习方式和资源是否充足？
   - 是否有导师制或辅导机制？

3. 人才梯队
   - 关键岗位是否有继任计划？
   - 高潜人才是否被识别和培养？
   - 人才流动和晋升通道是否畅通？

4. 知识管理
   - 经验和知识是否有效沉淀？
   - 是否有知识共享机制？
   - 新人能否快速上手？
```

#### 典型问题示例

```
【能力匹配】
- 你觉得胜任当前岗位最关键的 3 项能力是什么？
- 你觉得自己哪些方面已经达标？哪些方面还有差距？
- 工作中最常遇到的能力挑战是什么？

【培养发展】
- 过去一年你参加过哪些培训？哪些最有帮助？
- 当遇到新挑战时，你通常如何学习？
- 你希望公司提供什么样的发展支持？

【人才梯队】
- 如果有人接替你的工作，需要多久能上手？
- 你觉得团队中谁最有潜力？为什么？
- 你对自己的晋升路径清晰吗？

【知识管理】
- 新人加入团队通常如何学习？
- 团队有经验的同事离职会有什么影响？
- 有什么好的工作方法值得分享但还没沉淀下来？
```

#### 常见问题信号

| 信号 | 可能问题 | 追问方向 |
|-----|---------|---------|
| "没人教，自己摸索" | 培养体系缺失 | 了解学习资源和导师机制 |
| "关键人就那几个" | 人才梯队风险 | 诊断知识集中度和继任计划 |
| "培训都是走过场" | 培训有效性低 | 了解培训内容和方式 |
| "同样的错误反复犯" | 知识管理缺失 | 探索经验沉淀机制 |

---

### 三、员工治理（Governance）- "允不允许"

**诊断重点：** 组织架构、流程制度是否支持员工发挥能力

#### 关键诊断维度

```
1. 组织架构
   - 架构是否支持战略执行？
   - 职责分工是否清晰？
   - 汇报关系是否合理？

2. 流程制度
   - 核心流程是否高效？
   - 制度是否支持业务需求？
   - 是否存在冗余或僵化？

3. 授权机制
   - 员工是否有足够的决策权？
   - 审批流程是否过长？
   - 权责是否对等？

4. 信息平台
   - 信息是否透明和及时？
   - 协作工具是否有效？
   - 数据是否支持决策？
```

#### 典型问题示例

```
【组织架构】
- 你清楚自己的职责边界吗？
- 有没有遇到过"三不管"或"多人管"的事情？
- 你觉得团队架构有什么可以优化的地方？

【流程制度】
- 日常工作中哪些流程让你觉得效率低？
- 有没有"为了流程而流程"的情况？
- 如果让你 redesign 一个流程，你会改哪个？

【授权机制】
- 你工作中多大比例的事情可以自主决策？
- 一个典型决策需要经过多少人审批？
- 有没有因为等待审批而耽误工作的经历？

【信息平台】
- 你能及时获取工作需要的信息吗？
- 团队之间信息透明吗？
- 常用的协作工具好用吗？有什么痛点？
```

#### 常见问题信号

| 信号 | 可能问题 | 追问方向 |
|-----|---------|---------|
| "这事儿得问领导" | 授权不足 | 了解决策权限和审批流程 |
| "流程太复杂了" | 流程冗余 | 识别低效环节和优化点 |
| "信息不透明" | 沟通机制缺失 | 探索信息共享渠道 |
| "部门墙太厚" | 协作机制问题 | 了解跨部门协作障碍 |

## 🎯 组织诊断应用指南

### 诊断流程

```
1. 明确诊断目的
   ↓
2. 选择诊断维度（思维/能力/治理）
   ↓
3. 设计访谈问题（基于三大支柱）
   ↓
4. 收集信息（访谈/问卷/观察）
   ↓
5. 分析差距（现状 vs 期望）
   ↓
6. 制定改进计划（优先级排序）
```

### 维度选择逻辑

根据诊断场景和员工角色，选择重点诊断维度：

| 诊断场景 | 优先维度 | 说明 |
|---------|---------|------|
| **业绩下滑** | 思维 → 能力 → 治理 | 先检查意愿，再检查能力，最后检查机制 |
| **人才流失** | 思维 → 治理 → 能力 | 先了解离职动机，再检查管理机制 |
| **扩张期** | 能力 → 治理 → 思维 | 优先解决人才缺口，再优化架构 |
| **变革期** | 思维 → 治理 → 能力 | 先统一认知，再调整机制 |
| **创新乏力** | 治理 → 思维 → 能力 | 先松绑机制，再激活动机 |
| **执行不力** | 治理 → 能力 → 思维 | 先检查流程授权，再检查能力匹配 |

### 员工角色适配

| 员工角色 | 思维维度 | 能力维度 | 治理维度 |
|---------|---------|---------|---------|
| **高层管理者** | 战略认同、使命感 | 领导力、战略思维 | 决策机制、信息透明 |
| **中层管理者** | 目标对齐、责任感 | 管理能力、业务专业 | 授权机制、流程支持 |
| **一线员工** | 工作动机、归属感 | 岗位技能、学习能力 | 工作流程、工具支持 |
| **新员工** | 文化融入、期望管理 | 岗位胜任、快速学习 | 入职流程、导师机制 |
| **老员工** | 持续激励、意义感 | 知识更新、经验传承 | 参与决策、知识管理 |

## 📊 问题设计技巧

### 五层追问法（引导系统性思考）

```
【现象层】你遇到了什么具体问题？能举个例子吗？

【影响层】这个问题对你/团队的工作有什么影响？

【机制层】你觉得这个问题是个例，还是其他人也会遇到？

【系统层】如果是普遍问题，你觉得是什么机制/流程导致的？
         （对应杨三角：思维/能力/治理哪个环节的问题？）

【建议层】如果让你设计一个机制来避免这个问题，你会怎么做？
```

### 杨三角归因引导

当员工提出问题时，引导其归因到三大支柱：

```
【思维归因】
- 这个问题和大家的意愿/动机有关系吗？
- 是不是大家对目标的优先级理解不一致？

【能力归因】
- 这个问题需要什么样的能力才能解决？
- 团队是否具备这些能力？

【治理归因】
- 现有的流程/制度/授权是否支持解决这个问题？
- 是什么机制导致这个问题反复出现？
```

## 🔗 与韦斯伯德六盒模型的对比

| 维度 | 杨三角 | 韦斯伯德六盒 | 适用场景 |
|-----|-------|------------|---------|
| **框架复杂度** | 3 个支柱（简洁） | 6 个盒子（全面） | 杨三角适合快速诊断 |
| **关注焦点** | 员工视角（思维/能力/治理） | 组织视角（目的/结构/关系等） | 杨三角更贴近 HR 工作 |
| **问题归因** | 明确指向三大支柱 | 多维度交叉分析 | 杨三角更易制定行动计划 |
| **文化适配** | 源自中国企业实践 | 源自西方组织理论 | 杨三角更适合中国互联网公司 |

**建议：**
- **快速诊断/HR 主导** → 优先使用杨三角
- **全面诊断/外部顾问** → 可使用韦斯伯德六盒
- **两者结合** → 用杨三角归类问题，用六盒模型深入分析

## 📋 访谈提纲模板（杨三角版）

```markdown
# 组织诊断访谈提纲（杨三角框架）

## 📌 访谈基本信息
- 员工姓名：XXX
- 职位/职级：XXX
- 所属部门：XXX
- 访谈类型：组织诊断
- 重点维度：[员工思维/员工能力/员工治理]

## 🎯 诊断目的
1. 评估组织健康度
2. 识别关键能力差距
3. 收集改进建议

## 💬 访谈问题

### 一、员工思维（愿不愿意）
- 你觉得团队的目标有意义吗？
- 什么能让你在工作中保持动力？
- 你觉得付出和回报匹配吗？

### 二、员工能力（会不会）
- 你觉得自己胜任岗位的关键能力是什么？
- 哪些方面还需要提升？
- 公司提供的培养支持够吗？

### 三、员工治理（允不允许）
- 工作流程中有卡点吗？
- 你有足够的决策权吗？
- 信息透明和协作顺畅吗？

### 四、系统性追问
- 这个问题是个人问题还是机制问题？
- 如果是机制问题，属于思维/能力/治理哪个方面？
- 如果让你改进，你会怎么做？
```

## ⚠️ 使用注意事项

### 避免的误区

| 误区 | 正确做法 |
|-----|---------|
| 只诊断不行动 | 诊断后必须有改进行动计划 |
| 归因于个人 | 引导思考系统性原因 |
| 三大支柱割裂 | 关注支柱之间的相互影响 |
| 一次性诊断 | 定期复盘，持续追踪 |

### 敏感话题处理

- **思维问题**：避免质疑员工"态度不好"，聚焦"什么影响动机"
- **能力问题**：避免贴标签"能力不足"，聚焦"能力缺口和培养"
- **治理问题**：避免批评"制度僵化"，聚焦"如何优化支持业务"

### 保密原则

- 强调访谈内容保密
- 汇总分析时去标识化
- 敏感问题单独上报
