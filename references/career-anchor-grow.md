# 职业发展访谈指南 - 职业锚理论 + GROW 模型

本指南为使用职业锚理论和 GROW 模型进行职业发展访谈提供详细指导。

## 核心理论概述

### 1. 职业锚理论（Career Anchor Theory）

由 Edgar Schein 提出，指个人在职业选择中不会放弃的 8 种核心价值观：

- **技术/职能型**（Technical/Functional Competence）：追求专业技能精进
- **管理型**（Managerial Competence）：追求承担责任和领导团队
- **自主/独立型**（Autonomy/Independence）：追求工作自主性，不喜欢被管控
- **安全/稳定型**（Security/Stability）：追求稳定和可预测性，厌恶风险
- **创业/创造型**（Entrepreneurial Creativity）：追求创造新事物，喜欢冒险
- **服务/奉献型**（Service/Dedication to a Cause）：追求帮助他人和社会贡献
- **纯挑战型**（Pure Challenge）：追求解决难题和战胜对手
- **生活方式型**（Lifestyle）：追求工作与生活的平衡

### 2. GROW 模型

由 Sir John Whitmore 提出，用于教练式对话的四个步骤：

- **G (Goal)**：目标设定（你想达成什么？）
- **R (Reality)**：现状分析（你现在在哪里？）
- **O (Options)**：选择探索（你有什么选择？）
- **W (Will)**：意愿/行动（你将采取什么行动？）

## 8 种职业锚的详细应用

### 1. 技术/职能型

**核心特征**：追求专业技能精进，成为某个领域的专家

**访谈问题示例**：
- 你希望成为哪个领域的专家？
- 管理岗位对你有吸引力吗？为什么？
- 你觉得什么样的工作最能发挥你的专业能力？

**识别信号**：
- 喜欢技术挑战，不喜欢管理事务
- 关注技能提升，关注项目质量
- 对培训、学习机会敏感

### 2. 管理型

**核心特征**：追求承担责任和领导团队

**访谈问题示例**：
- 你更喜欢领导团队还是个人贡献？
- 你希望管理多大的团队？
- 什么样的管理方式最适合你？

**识别信号**：
- 喜欢协调资源、推动项目
- 关注团队整体目标
- 对晋升、决策权敏感

### 3. 自主/独立型

**核心特征**：追求工作自主性，不喜欢被管控

**访谈问题示例**：
- 你希望有多大的自主权？
- 什么样的工作方式让你最舒服？
- 你如何平衡自主性和团队协作？

**识别信号**：
- 喜欢灵活的工作安排
- 对流程和管控敏感
- 追求个人成就感

### 4. 安全/稳定型

**核心特征**：追求稳定和可预测性，厌恶风险

**访谈问题示例**：
- 你理想的工作状态是什么样的？
- 什么能让你感到安心和稳定？
- 你对风险的态度是什么？

**识别信号**：
- 关注工作稳定性
- 对变动、不确定性敏感
- 追求可预测的工作环境

### 5. 创业/创造型

**核心特征**：追求创造新事物，喜欢冒险

**访谈问题示例**：
- 你希望创造什么样的价值？
- 什么样的工作能让你感到兴奋？
- 你如何看待创新和风险？

**识别信号**：
- 喜欢尝试新事物
- 对创新、创业敏感
- 追求创造性的工作

### 6. 服务/奉献型

**核心特征**：追求帮助他人和社会贡献

**访谈问题示例**：
- 什么样的工作让你感到有意义？
- 你希望对社会做出什么贡献？
- 你的工作价值观是什么？

**识别信号**：
- 关注工作意义
- 对帮助他人、社会贡献敏感
- 追求价值实现

### 7. 纯挑战型

**核心特征**：追求解决难题和战胜对手

**访谈问题示例**：
- 什么样的挑战能让你保持激情？
- 你如何定义成功？
- 你希望在什么领域产生更大影响？

**识别信号**：
- 喜欢高难度任务
- 对挑战、成就敏感
- 追求突破性成果

### 8. 生活方式型

**核心特征**：追求工作与生活的平衡

**访谈问题示例**：
- 你理想的工作生活平衡是什么样的？
- 什么对你来说最重要？
- 工作在你生活中的位置是什么？

**识别信号**：
- 关注工作生活平衡
- 对工作压力、加班敏感
- 追求生活质量

## 职业发展访谈完整提纲

### 一、Goal - 目标设定（20分钟）

**目标**：帮助员工明确职业发展方向

**主要问题**：
1. 回顾过去 1-2 年，你最有成就感的时刻是什么？为什么？
2. 从这些经历中，你发现了什么驱动力？
3. 你未来 1 年、3 年、5 年希望达成什么样的职业目标？
4. 这些目标对你来说为什么重要？

**职业锚识别提示**：
- 技术型：关注技能深度、专业成就
- 管理型：关注领导力、影响力
- 自主型：关注工作自由度
- 安全型：关注稳定性、确定性
- 创业型：关注创新、创造
- 服务型：关注社会价值
- 挑战型：关注挑战、突破
- 生活方式型：关注工作生活平衡

### 二、Reality - 现状分析（20分钟）

**目标**：评估当前工作状态与目标的差距

**主要问题**：
1. 你对目前的工作满意度如何？1-10 分打个分
2. 哪些方面让你满意？哪些方面不满意？
3. 你的核心优势和短板是什么？
4. 要达成你的职业目标，你现在还需要具备什么？
5. 公司/团队能提供什么支持？还缺什么？

**现状分析要点**：
- 工作内容匹配度
- 能力成长空间
- 资源支持充足性
- 发展机会可见性

### 三、Options - 选择探索（15分钟）

**目标**：探讨职业发展路径和可能性

**主要问题**：
1. 要达成你的职业目标，有哪些可能的路径？
2. 内部有哪些发展机会（转岗、晋升、新项目等）？
3. 外部有哪些可能性（如果考虑跳槽，会关注什么）？
4. 每条路径的利弊是什么？
5. 你需要克服哪些障碍？

**路径类型**：
- **专业深度路径**：在当前领域深耕，成为专家
- **管理转型路径**：从个人贡献者转向管理岗位
- **跨领域发展路径**：转岗到其他领域，拓宽视野
- **内部创业路径**：在内部创新、创业
- **外部发展路径**：跳槽到其他公司

### 四、Will - 行动计划（15分钟）

**目标**：制定具体的行动计划

**主要问题**：
1. 在上述选项中，你倾向于选择哪条路径？为什么？
2. 接下来 3 个月，你计划采取什么具体行动？
3. 你需要公司、上级提供什么支持？
4. 我们如何衡量进展？什么时候回顾？
5. 如果遇到困难，你会如何应对？

**行动计划模板**：
| 时间 | 行动项 | 负责人 | 所需支持 | 预期成果 | 完成时间 |
|------|--------|--------|----------|----------|----------|
| 1个月 | | | | | |
| 3个月 | | | | | |
| 6个月 | | | | | |

### 五、总结与承诺（5分钟）

**目标**：确认要点，建立承诺

**要点复述**：
1. 复述员工的职业锚类型和发展目标
2. 确认行动计划和支持措施
3. 约定下次跟进时间

**承诺确认**：
"我理解你的职业锚是 [XX]，你希望在未来 [X 年] 达成 [XX 目标]。我们制定了 [XX] 的行动计划，公司会提供 [XX] 支持。我们将在 [X 月后] 回顾进展。"

## 产出物模板

### 职业发展计划（Markdown 格式）

```markdown
# 职业发展计划 - [员工姓名]

## 📌 基本信息

- **员工姓名**：张 XXX
- **当前职位**：后端开发工程师 P6
- **入职时间**：2023 年 3 月
- **访谈日期**：2025 年 3 月 12 日
- **访谈人**：HR 李 XXX

## 🎯 职业锚识别

**职业锚类型**：技术/职能型

**识别依据**：
- 最有成就感的时刻：高并发系统优化成功，支撑 10 倍流量增长
- 关注点：技能提升、技术挑战、项目质量
- 发展偏好：专业深度而非管理岗位

## 📊 职业目标

### 短期目标（1 年）
- 晋升至 P7，成为高并发系统专家
- 主导 2 个核心系统优化项目
- 掌握 Kubernetes 和云原生技术栈

### 中期目标（3 年）
- 晋升至 P8，成为技术专家
- 指导 3-5 名新人成长
- 在技术社区发表 2 篇技术文章

### 长期目标（5 年）
- 成为技术架构师或技术总监
- 在高并发、分布式系统领域建立影响力
- 培养一支技术过硬的团队

## 📋 GROW 分析表

### Goal（目标）
- 已明确短中长期目标
- 目标符合职业锚定位
- 目标具有挑战性和可达成性

### Reality（现状）
- **优势**：技术扎实、项目经验丰富、学习能力强
- **短板**：K8s 和云原生技术待提升、系统设计经验不足
- **支持**：公司有技术分享机制、导师制度
- **差距**：从 P6 到 P7 需要系统设计能力和技术影响力

### Options（选择）
- **路径 1**：当前岗位深耕 + 参与系统设计项目
- **路径 2**：转岗到架构组，专注系统架构
- **路径 3**：参与开源项目，提升技术影响力
- **选择**：路径 1（当前岗位深耕 + 系统设计项目）

### Will（行动）

#### 3 个月行动计划
1. 学习 K8s 官方文档和最佳实践（负责人：员工）
2. 参与系统架构设计评审会议（负责人：员工，支持：技术总监）
3. 完成 1 个技术分享主题（负责人：员工）
4. 安排技术导师（负责人：HR）

#### 6 个月行动计划
1. 主导 1 个微服务拆分项目（负责人：员工）
2. 发表 1 篇技术博客（负责人：员工）
3. 参与技术面试，提升技术影响力（负责人：技术总监）
4. 复盘进展，调整计划（负责人：员工、HR）

#### 12 个月行动计划
1. 晋升 P7 评审准备（负责人：员工、上级）
2. 指导 1 名新人（负责人：员工）
3. 参与技术架构决策（负责人：技术总监）
4. 评估是否达成短期目标（负责人：HR、上级）

## 🤝 支持措施

### 公司支持
- 提供技术培训和认证支持
- 安排技术导师指导
- 提供参与架构设计的机会
- 支持参加技术会议和社区活动

### 上级支持
- 每季度 1 对 1 沟通职业发展
- 提供有挑战的项目机会
- 给予系统设计决策权
- 鼓励技术分享和影响力建设

### HR 支持
- 定期跟进职业发展进展
- 协调培训和导师资源
- 提供晋升评估和反馈
- 组织职业发展规划讨论

## 📅 跟进计划

| 时间 | 跟进内容 | 参与人 |
|------|----------|--------|
| 3 个月后 | 回顾 3 个月目标达成情况 | 员工、HR、上级 |
| 6 个月后 | 调整中期行动计划 | 员工、HR、上级 |
| 12 个月后 | 评估短期目标达成情况，规划下一年 | 员工、HR、上级 |

## ⚠️ 注意事项

### 识别原则
- 职业锚需员工自我发现，基于经历，无好坏之分
- 同一员工可能有多个职业锚，以最突出的为主
- 职业锚可能随时间变化，需要定期回顾

### 使用技巧
- **多问少说**（80/20 法则）：员工多说，HR 引导
- **开放性问题**：避免是非题，鼓励深入思考
- **不评判**：尊重员工的职业锚选择
- **聚焦行动**：将讨论转化为具体行动计划

### 常见误区
- 避免直接给建议，而是引导员工自己发现
- 避免忽视职业锚匹配度，推动不合适的发展路径
- 避免设定过大目标，缺乏可操作性
- 避免缺乏跟进，计划无法落地
