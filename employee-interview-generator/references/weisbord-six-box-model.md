# 组织诊断访谈指南 - 韦斯伯德六个盒子模型

本指南为使用韦斯伯德六个盒子模型（Weisbord's Six-Box Model）进行组织诊断访谈提供详细指导。

---

## 📊 模型介绍

韦斯伯德六个盒子模型是由美国行为科学家 Marvin Weisbord 于 1976 年提出的组织诊断工具，从六个维度系统评估组织健康度。

```
┌─────────────────────────────────────────┐
│              外部环境                    │
│         (Customers, Market, etc.)       │
├─────────────┬─────────────┬─────────────┤
│   目的      │   结构      │   关系      │
│  (Purpose)  │ (Structure) │(Relationships)│
│  目标清晰吗？│  分工合理吗？│  协作顺畅吗？│
├─────────────┼─────────────┼─────────────┤
│   奖励      │   领导      │  支持机制   │
│  (Rewards)  │ (Leadership)│(Mechanisms) │
│  激励有效吗？│  方向明确吗？│  工具到位吗？│
├─────────────┴─────────────┴─────────────┤
│              内部协调                    │
└─────────────────────────────────────────┘
```

---

## 🎯 六个维度详解

### 1️⃣ 目的（Purpose）

**定义：** 组织目标是否清晰、被理解、被认同

**诊断问题：**
- 目标是否清晰传达？
- 员工是否理解并认同目标？
- 目标是否与日常工作对齐？

**常见问题信号：**
- ⚠️ 员工不清楚团队目标
- ⚠️ 目标频繁变化且沟通不足
- ⚠️ 个人目标与组织目标脱节

**系统性问题 vs 个人抱怨：**
| 个人抱怨 | 系统性问题 |
|---------|-----------|
| "我不知道目标是什么" | 目标传达机制缺失或不清晰 |
| "目标变来变去" | 缺乏目标变更管理和沟通流程 |
| "目标跟我没关系" | 目标分解和对齐机制缺失 |

---

### 2️⃣ 结构（Structure）

**定义：** 组织架构、分工、流程是否支持目标实现

**诊断问题：**
- 职责分工是否清晰？
- 流程是否顺畅高效？
- 架构是否支持战略？

**常见问题信号：**
- ⚠️ 职责边界模糊，存在"三不管"地带
- ⚠️ 跨部门协作困难，流程冗长
- ⚠️ 架构层级过多，决策缓慢

**系统性问题 vs 个人抱怨：**
| 个人抱怨 | 系统性问题 |
|---------|-----------|
| "我不知道这事该谁管" | 职责定义和 RACI 矩阵缺失 |
| "流程太复杂" | 流程设计未随业务变化迭代 |
| "决策太慢" | 决策权限不清晰或缺乏授权机制 |

---

### 3️⃣ 关系（Relationships）

**定义：** 人际关系、团队协作、冲突处理

**诊断问题：**
- 团队内部协作是否顺畅？
- 冲突是否有健康的处理机制？
- 跨团队协作是否有效？

**常见问题信号：**
- ⚠️ 团队内部沟通不畅，信息孤岛
- ⚠️ 冲突被回避或升级为个人恩怨
- ⚠️ 跨团队目标不一致，资源竞争

**系统性问题 vs 个人抱怨：**
| 个人抱怨 | 系统性问题 |
|---------|-----------|
| "XX 不配合我" | 缺乏跨团队协作机制和共同目标 |
| "开会都在吵架" | 缺乏有效的冲突解决机制和会议规范 |
| "他们不告诉我们" | 信息共享机制缺失或文化问题 |

---

### 4️⃣ 奖励（Rewards）

**定义：** 激励机制是否公平、有效、与目标对齐

**诊断问题：**
- 激励是否与贡献匹配？
- 激励是否与目标对齐？
- 激励方式是否多样化？

**常见问题信号：**
- ⚠️ 付出与回报不匹配感普遍
- ⚠️ 激励与目标脱节（如鼓励创新但惩罚失败）
- ⚠️ 激励方式单一，忽视非物质激励

**系统性问题 vs 个人抱怨：**
| 个人抱怨 | 系统性问题 |
|---------|-----------|
| "我干得多拿得少" | 绩效评估标准模糊或不公平 |
| "干好干坏一个样" | 激励机制缺乏差异化 |
| "只画饼不给钱" | 激励组合设计不合理 |

---

### 5️⃣ 领导（Leadership）

**定义：** 领导力、决策、方向指引

**诊断问题：**
- 领导是否提供清晰方向？
- 决策是否透明有效？
- 领导是否提供足够支持？

**常见问题信号：**
- ⚠️ 领导方向不清晰，频繁变化
- ⚠️ 决策黑箱，员工不参与
- ⚠️ 领导微观管理或缺位

**系统性问题 vs 个人抱怨：**
| 个人抱怨 | 系统性问题 |
|---------|-----------|
| "领导不说清楚要什么" | 缺乏目标传达和反馈机制 |
| "决策不透明" | 缺乏决策流程和沟通机制 |
| "领导不管我们" | 管理幅度太大或缺乏 1 对 1 机制 |

---

### 6️⃣ 支持机制（Support Mechanisms）

**定义：** 工具、流程、信息、培训等支持是否到位

**诊断问题：**
- 工具是否支持高效工作？
- 信息是否透明及时？
- 培训是否满足成长需求？

**常见问题信号：**
- ⚠️ 工具落后，效率低下
- ⚠️ 信息不透明，决策缺乏依据
- ⚠️ 培训不足，能力跟不上

**系统性问题 vs 个人抱怨：**
| 个人抱怨 | 系统性问题 |
|---------|-----------|
| "系统太难用" | 缺乏工具评估和迭代机制 |
| "什么都不知道" | 信息共享机制缺失 |
| "没人教我" | 缺乏能力模型和培训体系 |

## 🔍 维度选择算法

根据员工角色自动选择最相关的 3-6 个维度：

```python
def select_dimensions(employee):
    """
    根据员工角色选择诊断维度
    """
    dimensions = {
        '高管': ['目的', '领导', '结构'],
        '中层': ['结构', '关系', '支持机制'],
        '一线': ['奖励', '支持机制', '关系'],
        '技术': ['支持机制', '目的', '奖励'],
        '新员工': ['结构', '关系', '支持机制'],
        '老员工': ['目的', '奖励', '领导'],
    }
    
    role = identify_role(employee)
    return dimensions.get(role, ['目的', '结构', '关系', '奖励', '领导', '支持机制'])
```

## 💬 追问技巧：引导系统性思考

### 核心原则

1. **从具体到抽象** - 先问具体事例，再问系统原因
2. **从个人到组织** - 先问个人体验，再问组织机制
3. **从现象到本质** - 先问表面现象，再问深层原因

### 追问模板

```
【现象层】你遇到了什么具体问题？能举个例子吗？

【影响层】这个问题对你/团队的工作有什么影响？

【机制层】你觉得这个问题是个例，还是其他人也会遇到？

【系统层】如果是普遍问题，你觉得是什么机制/流程导致的？

【建议层】如果让你设计一个机制来避免这个问题，你会怎么做？
```

### 避免的陷阱

| 陷阱 | 错误示例 | 正确示例 |
|-----|---------|---------|
| **引导性提问** | "你是不是觉得领导不行？" | "你和上级的沟通怎么样？" |
| **过早下结论** | "这就是流程问题" | "你觉得可能是什么原因？" |
| **忽视情绪** | "别抱怨了，说重点" | "我理解这让你很沮丧，能具体说说吗？" |
| **个人化问题** | "你为什么做不好？" | "是什么阻碍了你做好？" |

## 📝 访谈后分析框架

### 问题分类矩阵

```
                影响范围
              小 ←───→ 大
            ┌───────┬───────┐
         低 │ 个人  │ 流程  │
        频  │ 问题  │ 优化  │
    发    ├───────┼───────┤
    生    │ 团队  │ 系统  │
    频    │ 协调  │ 变革  │
         高 └───────┴───────┘
```

### 问题优先级

| 类型 | 特征 | 处理方式 |
|-----|------|---------|
| **个人问题** | 低频、小范围 | 个人辅导/调解 |
| **流程优化** | 高频、小范围 | 流程改进 |
| **团队协调** | 低频、大范围 | 团队对齐/机制建立 |
| **系统变革** | 高频、大范围 | 组织级变革 |

## ⚠️ 注意事项

### 访谈伦理

1. **保密原则** - 不透露具体访谈对象
2. **中立立场** - 不评判、不站队
3. **知情同意** - 告知访谈目的和数据使用
4. **心理安全** - 允许匿名、允许跳过敏感问题

### 常见偏差

| 偏差类型 | 表现 | 避免方法 |
|---------|------|---------|
| **近因效应** | 过度关注最近事件 | 询问整个周期的体验 |
| **从众效应** | 迎合主流观点 | 强调需要多元声音 |
| **社会期许** | 说"正确"答案 | 强调没有标准答案 |
| **负面偏好** | 聚焦问题忽视优点 | 主动询问积极方面 |
