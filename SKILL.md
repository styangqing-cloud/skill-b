# Employee Interview Generator - 员工访谈提纲生成器

专为互联网公司 HR 设计的员工访谈提纲生成技能。

## 🎯 何时使用

**当 HR 需要进行以下类型的员工访谈时触发此技能：**

| 访谈类型 | 触发场景 | 典型时机 |
|---------|---------|---------|
| **入职访谈** | 新员工融入、试用期跟进 | 入职 1 周/1 月/3 月 |
| **绩效访谈** | 绩效评估、目标设定 | 季度/年度绩效周期 |
| **离职访谈** | 员工离职原因了解 | 提交离职申请后 |
| **晋升访谈** | 晋升评估、职级面谈 | 晋升评审前/后 |
| **职业发展** | IDP 制定、职业规划 | 年度/半年度 |
| **团队反馈** | 团队氛围、管理反馈 | 不定期/项目后 |
| **挽留访谈** | 核心员工挽留 | 发现离职倾向时 |
| **转岗访谈** | 内部转岗评估 | 转岗申请后 |

**关键触发短语：**
- "帮我生成一个员工访谈提纲"
- "准备和 XXX 做个访谈"
- "需要和员工做个 1 对 1"
- "准备个绩效面谈提纲"
- "做个离职访谈准备"

## 📋 工作流程

```
1️⃣ 收集员工信息（文件/链接/手动） → 2️⃣ 分析呈现员工信息 → 3️⃣ 生成访谈思路（≤300 字） → 4️⃣ 用户确认 → 5️⃣ 生成完整提纲 → 6️⃣ 输出/优化
```

### 阶段 1：收集员工信息

**三种信息收集方式：**

#### 方式 A：上传文件（推荐）

**支持的文件格式：**
| 格式 | 适用场景 | 说明 |
|-----|---------|------|
| **Excel** | 绩效数据、考勤记录、项目清单 | 自动解析表格数据 |
| **PDF** | 简历、绩效报告、评估文档 | 提取文本内容 |
| **Word** | 工作总结、述职报告 | 提取文档内容 |
| **PPTX** | 述职报告、项目汇报 | 提取幻灯片内容 |
| **TXT** | 纯文本记录、笔记 | 直接读取 |

**处理方式：**
1. 用户上传文件
2. 调用相应工具解析文件内容（document-handler / document-parser / tencent-docs.get_content）
3. 提取员工相关信息

#### 方式 B：网页链接

**适用的链接类型：**
- 公司内部系统链接（如绩效系统、项目管理系统）
- 公开的个人主页（如 GitHub、LinkedIn、博客）
- 文档链接（如腾讯文档、飞书文档）

**处理方式：**
1. 用户提供可访问的网页链接
2. 调用 web_fetch 或 browser 工具抓取页面内容
3. 提取员工相关信息

#### 方式 C：手动输入

HR 直接提供员工信息，适用于：
- 没有现成文件/链接
- 信息较少或简单场景
- 紧急需要提纲

---

**必须收集的核心信息：**

```yaml
基本信息:
  - 姓名（可用"某员工"代替）
  - 职位/职级
  - 所属部门
  - 入职时间
  - 汇报对象

背景信息（可选但推荐）:
  - 过往绩效表现
  - 当前项目/职责
  - 特殊情况（如近期表现变化、团队调整等）
  - 历史访谈记录（如有）
```

**信息收集方式：**
- 文件/链接自动解析 → 提取并确认
- HR 主动提供 → 直接记录
- HR 不确定 → 提供信息收集清单让其补充
- 部分信息缺失 → 基于已有信息生成，标注待确认项

<HARD-GATE>
必须收集到以下核心信息才能生成提纲：
1. 访谈类型（8 种之一）
2. 员工职级/职位
3. 访谈目的（至少 1 个）

缺少任一信息时，必须向 HR 追问，不能直接生成。
</HARD-GATE>

### 阶段 1.5：分析并呈现员工信息

**文件/链接解析后，必须执行以下步骤：**

1. **信息提取** - 从文件/链接中提取员工相关信息
2. **信息分类** - 按基本信息、绩效表现、项目经历、能力特长等分类
3. **信息呈现** - 以结构化方式向 HR 呈现分析结果
4. **信息确认** - HR 确认信息准确性，补充或修正

**呈现格式示例：**

```markdown
## 📊 员工信息分析结果

### 基本信息
- 姓名：张 XXX
- 职位：后端开发工程师 P6
- 部门：技术部 - 基础架构组
- 入职时间：2023 年 3 月（2 年）

### 绩效表现
- 2024 Q1-Q4：连续 4 季度超出预期
- 年度评级：S（前 10%）

### 核心项目
- 项目 A：负责 XX 模块，提升性能 30%
- 项目 B：主导 XX 系统重构

### 能力特长
- 技术栈：Go/Python/K8s
- 特长：高并发系统设计

### 特殊情况
- 近期工作负荷较大
- 有晋升意向
```

**注意：** 如信息不足或有疑问，需向 HR 确认后再继续。

---

### 阶段 2：生成个性化访谈思路（≤300 字）

**在生成完整提纲前，必须先生成并确认访谈思路。**

**访谈思路内容：**
1. **访谈重点** - 基于员工信息，说明本次访谈的核心关注点
2. **问题策略** - 说明问题设计的侧重点和方式
3. **注意事项** - 提醒需要特别注意的敏感点或风险

**格式要求：**
- 不超过 300 字
- 简洁清晰，重点突出
- 便于 HR 快速理解访谈策略

**示例：**

```
【访谈思路】

本次绩效访谈聚焦三大重点：1）肯定连续高绩效表现，强化成就感；2）了解高负荷下的工作状态，评估 burnout 风险；3）探索晋升诉求，明确发展路径。

问题策略：以开放式问题为主，先肯定成绩建立信任，再深入探讨工作状态和发展诉求。避免直接询问"是否考虑离职"，而是通过"工作节奏感受"间接了解。

注意事项：员工近期工作负荷大，避免过度施压；有晋升意向但本期名额有限，需谨慎回应；如表露离职倾向，立即启动挽留评估。
```

<HARD-GATE>
必须等待 HR 确认访谈思路后，才能生成完整提纲。
HR 可要求调整思路，确认后再继续。
</HARD-GATE>

### 阶段 2：确认访谈类型

**8 种访谈类型详解：**

| 类型 | 核心目的 | 问题重点 | 时长建议 |
|-----|---------|---------|---------|
| 入职访谈 | 融入情况、困难识别 | 适应度、支持需求、期望对齐 | 30-45min |
| 绩效访谈 | 目标达成、改进计划 | 成果回顾、差距分析、下阶段目标 | 45-60min |
| 离职访谈 | 离职原因、改进建议 | 真实原因、团队反馈、交接安排 | 30-60min |
| 晋升访谈 | 能力评估、期望沟通 | 能力证明、角色理解、发展计划 | 45-60min |
| 职业发展 | 成长路径、IDP 制定 | 职业目标、能力差距、资源需求 | 60min |
| 团队反馈 | 氛围诊断、管理改进 | 团队协作、管理反馈、改进建议 | 30-45min |
| 挽留访谈 | 离职风险、挽留方案 | 核心诉求、挽留条件、替代方案 | 45-60min |
| 转岗访谈 | 转岗动机、适配评估 | 转岗原因、能力匹配、过渡计划 | 30-45min |

### 阶段 3：明确访谈目的

**常见访谈目的库：**

```
入职访谈目的：
- 了解适应情况
- 识别融入障碍
- 明确期望对齐
- 建立信任关系

绩效访谈目的：
- 回顾目标达成
- 识别改进空间
- 设定下阶段目标
- 讨论发展需求

离职访谈目的：
- 了解真实原因
- 收集改进建议
- 评估挽留可能
- 确保平稳交接

...（其他类型类似）
```

**目的澄清方法：**
- HR 主动说明 → 记录并确认
- HR 模糊表达 → 提供选项让其选择
- 多目的场景 → 按优先级排序，建议聚焦 1-2 个核心目的

### 阶段 4：生成提纲

**提纲标准结构：**

```markdown
# 员工访谈提纲

## 📌 访谈基本信息
- 员工姓名：XXX
- 职位/职级：XXX
- 所属部门：XXX
- 入职时间：XXX
- 访谈类型：XXX
- 预计时长：XXX
- 访谈人：HR XXX
- 访谈日期：YYYY-MM-DD

## 🎯 访谈目的
1. [核心目的 1]
2. [核心目的 2]
3. ...

## 🔍 背景信息
- [员工当前状态/特殊情况]
- [过往相关记录]
- [需要注意的敏感点]

## 💬 访谈提纲

### 一、开场暖场（5 分钟）
[建立信任、缓解紧张的问题]

### 二、核心问题（按主题分类）

#### 主题 1：XXX
- 主问题：...
  - 追问提示：...
  - 追问提示：...
- 主问题：...
  - 追问提示：...

#### 主题 2：XXX
...

### 三、敏感/深度问题（谨慎使用）
[根据访谈进展选择性使用]

### 四、员工提问环节
[预留员工提问空间]

### 五、总结与后续行动
- 关键要点总结
- 双方约定的后续行动
- 下次跟进时间

## 📝 访谈记录区
[预留记录空间]

## ⚠️ 注意事项
- [针对该访谈的特殊提醒]
- [敏感话题处理建议]
- [风险预警]
```

**问题设计原则：**

| 原则 | 说明 | 示例 |
|-----|------|-----|
| 开放式优先 | 避免是/否回答 | "你觉得...怎么样？"vs"你满意吗？" |
| 由浅入深 | 先易后难，建立信任 | 先问适应情况，再问敏感问题 |
| 具体化 | 避免抽象，要求举例 | "能举个具体例子吗？" |
| 中性表述 | 避免引导性提问 | "发生了什么"vs"是不是因为..." |
| 追问设计 | 每个主问题配 2-3 个追问 | 深挖细节，获取真实信息 |

### 阶段 5：输出与优化

**输出格式选项：**
1. **Markdown 文本** - 直接输出，便于复制
2. **腾讯文档** - 调用 tencent-docs 技能创建在线文档
3. **Word 文档** - 调用 document-processor 技能

**优化迭代：**
- HR 反馈调整 → 根据反馈修改提纲
- 增加/删除问题 → 灵活调整
- 调整语气风格 → 正式/轻松/关怀等

## 📚 问题库（按访谈类型）

### 入职访谈问题库

```
适应情况：
- 入职这几周感觉怎么样？
- 实际工作内容和面试时了解的一致吗？
- 团队氛围符合你的预期吗？

支持需求：
- 有什么地方觉得不太顺手的吗？
- 需要我或团队提供什么支持？
- 有没有遇到什么困难？

期望对齐：
- 你对这个岗位的期望是什么？
- 你希望在这个岗位上获得什么成长？
- 有什么想了解的公司发展信息吗？
```

### 绩效访谈问题库

```
目标回顾：
- 上季度/年度你给自己打几分？为什么？
- 你觉得最重要的成就是什么？
- 有什么遗憾或觉得可以做得更好的地方？

差距分析：
- 目标达成过程中遇到的最大挑战是什么？
- 哪些因素影响了你的表现？
- 如果重来一次，你会怎么做？

发展计划：
- 下阶段你最想提升的能力是什么？
- 需要什么资源或支持？
- 你的职业发展目标是什么？
```

### 离职访谈问题库

```
离职原因：
- 能分享一下做出这个决定的主要原因吗？
- 是什么让你开始考虑离开的？
- 有没有什么是我们本可以做但没做的？

团队反馈：
- 在团队工作的这段时间，感受如何？
- 对管理方式有什么建议吗？
- 团队协作方面有什么想说的？

改进建议：
- 如果给公司提三个改进建议，会是什么？
- 有什么是我们应该保持的？
- 对接班的同事有什么建议吗？
```

### 晋升访谈问题库

```
能力证明：
- 你觉得哪些成果证明了你达到了下一职级的要求？
- 能分享一个你最有代表性的项目吗？
- 在这个过程中你展现了哪些核心能力？

角色理解：
- 你如何理解下一职级的职责和要求？
- 晋升后你计划如何开展工作？
- 你觉得自己还有哪些需要提升的地方？

发展计划：
- 晋升后你的工作重点会是什么？
- 短期和长期目标分别是什么？
- 需要什么支持？
```

## ⚠️ 注意事项

### 敏感话题处理

| 敏感话题 | 处理建议 |
|---------|---------|
| 薪酬不满 | 先倾听，不承诺，记录后反馈 |
| 上级矛盾 | 保持中立，了解事实，不站队 |
| 离职倾向 | 探索真实原因，评估挽留可能 |
| 绩效争议 | 基于事实，提供申诉渠道 |
| 职场骚扰 | 严肃对待，按流程上报 |

### 风险预警

**高风险信号（需特别关注）：**
- 员工表达强烈不满
- 提及法律/合规问题
- 情绪明显异常
- 涉及多人/团队问题
- 可能引发劳动争议

**处理方式：**
1. 保持冷静，继续倾听
2. 不做承诺，不承认过错
3. 详细记录，及时上报
4. 必要时暂停访谈，寻求支持

### 文化适配

**互联网公司特点：**
- 语气可轻松但保持专业
- 鼓励开放表达
- 重视员工体验
- 关注成长与发展
- 扁平化沟通风格

## 📁 输出示例

### 示例 1：绩效访谈提纲

```markdown
# 员工访谈提纲 - 绩效面谈

## 📌 访谈基本信息
- 员工姓名：张 XXX
- 职位/职级：后端开发工程师 P6
- 所属部门：技术部 - 基础架构组
- 入职时间：2023 年 3 月
- 访谈类型：绩效访谈
- 预计时长：45-60 分钟
- 访谈人：HR 李 XXX
- 访谈日期：2025 年 Q1 绩效周期

## 🎯 访谈目的
1. 回顾 Q4 绩效目标达成情况
2. 识别工作中的挑战和改进空间
3. 讨论 2025 年个人发展计划
4. 了解资源和支持需求

## 🔍 背景信息
- Q4 绩效自评：超出预期
- 主管评价：整体表现优秀，技术能力强
- 特殊情况：参与了 2 个核心项目，工作负荷较大
- 关注点：工作生活平衡、职业发展路径

## 💬 访谈提纲

### 一、开场暖场（5 分钟）
- 最近工作节奏怎么样？还适应吗？
- 刚过去的这个季度，整体感觉如何？
- 今天想和你聊聊绩效和发展的话题，大概需要 45 分钟左右，过程中有任何想法都可以随时说~

### 二、核心问题

#### 主题 1：绩效回顾（15 分钟）
- 主问题：回顾 Q4 的几个目标，你觉得自己完成得怎么样？
  - 追问：哪个目标完成得最满意？为什么？
  - 追问：有没有哪个目标觉得比较吃力？
  - 追问：如果给自己打分，1-10 分你会打几分？

- 主问题：这个季度你最有成就感的工作是什么？
  - 追问：能具体讲讲你在这个项目中的贡献吗？
  - 追问：这个成果对团队/业务有什么价值？

#### 主题 2：挑战与改进（10 分钟）
- 主问题：工作中遇到的最大挑战是什么？
  - 追问：你是如何应对的？
  - 追问：如果再来一次，会有什么不同做法？

- 主问题：有什么地方觉得可以做得更好吗？
  - 追问：具体是哪些方面？
  - 追问：需要什么支持来改进？

#### 主题 3：职业发展（10 分钟）
- 主问题：你对自己的职业发展有什么规划？
  - 追问：短期（1 年）和长期（3 年）目标分别是什么？
  - 追问：觉得目前和下一个职级的差距在哪里？

- 主问题：你希望公司提供什么样的发展支持？
  - 追问：培训、项目机会、导师指导等，哪个对你最有帮助？

#### 主题 4：工作状态（5 分钟）
- 主问题：最近的工作节奏感觉怎么样？
  - 追问：工作和生活的平衡还好吗？
  - 追问：有什么我们可以帮你改善的吗？

### 三、员工提问环节（5 分钟）
- 你有什么想问我的吗？
- 对公司、团队、制度等方面有什么想了解的吗？

### 四、总结与后续行动（5 分钟）
- 总结今天讨论的关键要点
- 确认双方约定的后续行动
- 约定下次跟进时间

## 📝 访谈记录区

## ⚠️ 注意事项
- 员工近期工作负荷较大，关注其工作状态
- 职业发展是关注重点，准备职级晋升标准信息
- 如有薪酬相关问题，记录后反馈，不当场承诺
```
