# Employee Interview Generator - 员工访谈提纲生成器

专为互联网公司 HR 设计的访谈/面试提纲生成技能。**支持员工访谈和招聘面试两种场景**。

## 🎯 何时使用

### 场景 A：员工访谈（内部员工）

**当 HR 需要进行以下类型的员工访谈时触发此技能：**

| 访谈类型 | 触发场景 | 典型时机 |
|---------|---------|---------|
| **入职访谈** | 新员工融入、试用期跟进 | 入职 1 周/1 月/3 月 |
| **绩效访谈** | 绩效评估、目标设定 | 季度/年度绩效周期 |
| **离职访谈** | 员工离职原因了解 | 提交离职申请后 |
| **晋升访谈** | 晋升评估、职级面谈 | 晋升评审前/后 |
| **职业发展** | IDP 制定、职业规划 | 年度/半年度 |
| **团队反馈** | 团队氛围、管理反馈 | 不定期/项目后 |
| **挽留访谈** | 核心员工挽留 | 发现离职倾向时 |
| **转岗访谈** | 内部转岗评估 | 转岗申请后 |
| **组织诊断** ⭐ | 组织健康度评估、变革前诊断 | 组织调整/变革前、年度诊断 |

### 场景 B：招聘面试（外部候选人）⭐ 新增

**当 HR/面试官需要进行招聘面试时触发此技能：**

| 面试类型 | 触发场景 | 典型时机 |
|---------|---------|---------|
| **简历筛选后** | 初面/业务面准备 | 简历通过筛选 |
| **终面前** | 高管面/HR 面准备 | 进入最终轮次 |
| **校招面试** | 校园招聘批量面试 | 春秋招期间 |
| **社招面试** | 社会招聘面试 | 全年 |
| **内推面试** | 内部推荐面试 | 内推简历提交后 |

**关键触发短语：**

## 📋 工作流程

```
1️⃣ 收集员工信息（文件/链接/手动） → 2️⃣ 分析呈现员工信息 → 3️⃣ 生成访谈思路（≤300 字） → 4️⃣ 用户确认 → 5️⃣ 生成完整提纲 → 6️⃣ 输出/优化
```

### 阶段 1：收集员工信息

**三种信息收集方式：**

#### 方式 A：上传文件（推荐）

**支持的文件格式：**
| 格式 | 适用场景 | 说明 |
|-----|---------|------|
| **Excel** | 绩效数据、考勤记录、项目清单 | 自动解析表格数据 |
| **PDF** | 简历、绩效报告、评估文档 | 提取文本内容 |
| **Word** | 工作总结、述职报告 | 提取文档内容 |
| **PPTX** | 述职报告、项目汇报 | 提取幻灯片内容 |
| **TXT** | 纯文本记录、笔记 | 直接读取 |

**处理方式：**
1. 用户上传文件
2. 调用相应工具解析文件内容（document-handler / document-parser / tencent-docs.get_content）
3. 提取员工相关信息

#### 方式 B：网页链接

**适用的链接类型：**
- 公司内部系统链接（如绩效系统、项目管理系统）
- 公开的个人主页（如 GitHub、LinkedIn、博客）
- 文档链接（如腾讯文档、飞书文档）

**处理方式：**
1. 用户提供可访问的网页链接
2. 调用 web_fetch 或 browser 工具抓取页面内容
3. 提取员工相关信息

#### 方式 C：手动输入

HR 直接提供员工信息，适用于：
- 没有现成文件/链接
- 信息较少或简单场景
- 紧急需要提纲

  - 汇报对象

职级体系:
  专业序列（P 序列）:
    低职级：P4-P7（初级/高级专员/工程师）
    中职级：P8-P9（资深专家/经理）
    高职级：P10-P11（总监/高级总监）
    专家级：P12+（研究员/资深专家）
  
  管理序列（M 序列）:
    低职级：M4-M7（主管/经理）
    中职级：M8-M9（高级经理/总监）

  销售序列（S 序列）:
    低职级：S4-S7（销售代表/高级代表）
    中职级：S8-S9（销售经理/大区经理）
    高职级：S10-S11（销售总监/事业部总经理）
    专家级：S12+（销售 VP/首席销售官）
  
  设计序列（D 序列）:
    低职级：D4-D7（设计师/高级设计师）
    中职级：D8-D9（资深设计师/设计专家）
    高职级：D10-D11（设计总监/高级总监）
    专家级：D12+（首席设计师/设计研究员）
  
  管理职级（L 序列）:
    L1-1 副组长：基层管理，协助组长管理小团队（3-5 人）
    L1-2 组长：基层管理，独立管理小团队（5-10 人）
    L2-1 副总监：中层管理，管理多个小组（10-30 人）
    L2-2 总监：中层管理，管理大团队或核心业务（30-100 人）
    L2-3 高级总监：高层管理，管理事业部/大区（100 人+）
  
  员工类型识别:
    团队管理者：有管理职级（L1-1/L1-2/L2-1/L2-2/L2-3），带团队，对团队绩效负责
    一线员工：无管理职级，不带团队，对个人绩效负责

绩效信息:
  绩效等级:
    高绩效：Outstanding（超出预期，前 20%）
    中绩效：Good（符合预期，中间 70%）
    低绩效：Underperform（低于预期，后 10%）
  
  梯队排名:
    前 5%: 团队核心骨干，至关重要（重点保留）
    前 20%: 重要成员，表现优秀（激励培养）
    中间 50%: 稳定贡献者，中坚力量（维持发展）
    后 20%: 表现一般，需关注改进（改进或调整）
    末 5%: 表现不佳，需淘汰（淘汰或转岗）
  
  绩效评语:
    近 1 次绩效：⭐⭐⭐⭐⭐ 权重 70%，最重要参考
    近 2 次绩效：⭐⭐⭐ 权重 30%，辅助参考（识别趋势）
    
    评语提取要点:
      - 优势与成就（主要成果、核心优势）
      - 改进空间（提升方向、管理者期望）
      - 关键词（高频词、情感倾向、具体事例）
      - 变化趋势（与近 2 次对比，上升/下降/稳定）

背景信息（可选但推荐）:
  - 过往绩效表现
  - 当前项目/职责
  - 特殊情况（如近期表现变化、团队调整等）
  - 历史访谈记录（如有）

    - 职级（P5/P6/P7/M1/M2...）
    - 级别描述（初级/中级/高级/专家）

    - 核心能力维度（3-5 个）

    - 30 分钟/45 分钟/60 分钟
  
  面试形式:
    - 线下面试/视频面试/电话面试
    - 1 对 1/多对 1/群面
```

**招聘面试的方法论框架：**

| 方法论 | 用途 | 面试维度 |
|--------|------|---------|
| **STAR 面试法** | 行为面试提问 | 情境 (S) + 任务 (T) + 行动 (A) + 结果 (R) |
| **冰山模型** | 胜任力评估 | 知识技能 + 自我概念 + 特质 + 动机 |
| **BEM 模型** | 经历验证 | 背景 + 行为 + 结果 |
| **职业锚理论** | 动机匹配 | 职业定位 + 价值观 + 发展诉求 |
| **文化匹配度** | 组织适配 | 价值观 + 工作风格 + 团队融合 |

**面试问题设计原则：**

1. **STAR 原则**：每个问题都引导候选人描述具体情境、任务、行动、结果
2. **行为导向**：关注过去的行为，而非假设性问题
3. **量化验证**：追问具体数字、指标、成果
4. **红线索引**：识别虚假陈述、过度包装、价值观不匹配

**面试流程设计（标准结构）：**

```
1️⃣ 开场破冰（3-5 分钟）→ 2️⃣ 候选人自我介绍（3 分钟）→ 3️⃣ 核心能力考察（40-45 分钟）→ 4️⃣ 候选人提问（5-10 分钟）→ 5️⃣ 收尾说明（2-3 分钟）
```

**流程说明：**
| 环节 | 时长 | 目的 | 关键动作 |
|------|------|------|---------|
| **开场破冰** | 3-5 分钟 | 缓解紧张、建立信任 | 自我介绍、说明流程、轻松话题 |
| **候选人介绍** | 3 分钟 | 了解表达能力、职业脉络 | 请候选人做自我介绍 |
| **核心能力考察** | 40-45 分钟 | 评估能力匹配度 | STAR 行为面试、追问验证 |
| **候选人提问** | 5-10 分钟 | 了解关注点、双向选择 | 回答候选人问题 |
| **收尾说明** | 2-3 分钟 | 管理期望、后续安排 | 说明流程、时间节点 |

**提纲生成流程：**

```
1️⃣ 收集信息（简历+JD） → 2️⃣ 候选人画像分析 → 3️⃣ 生成提纲思路（≤300 字） → 4️⃣ 用户确认/优化 → 5️⃣ 生成完整提纲 → 6️⃣ 输出/优化
```

**阶段 2：候选人画像分析（新增）⭐**

基于简历信息，梳理候选人完整画像：
- 教育背景分析（学校/专业/GPA）
- 工作经历梳理（公司/职位/职责/成就）
- 项目经验提炼（角色/规模/成果）
- 技能清单整理（硬技能/软技能）
- 跳槽动机分析（推力/拉力）
- 与岗位匹配度评估

**阶段 3：提纲思路确认（新增）⭐**

在生成完整面试提纲前，先输出整体思路（≤300 字）：
- 考察重点（3-5 个核心维度）
- 问题设计策略（STAR 问题分布）
- 风险提示（需重点验证的点）
- 时间分配建议

**等待用户确认：**
- "以上是面试提纲的整体思路，是否需要调整？"
- "确认无误后，我将生成完整的面试提纲"

**阶段 4：生成完整提纲**

用户确认后，生成包含以下模块的完整提纲：
1. 开场破冰问题
2. 候选人自我介绍引导
3. 核心能力考察问题（STAR）
4. 候选人提问准备
5. 收尾说明要点
6. 面试评估表

```

### 阶段 1.5：分析并呈现员工信息

**文件/链接解析后，必须执行以下步骤：**

1. **信息提取** - 从文件/链接中提取员工相关信息
2. **信息分类** - 按基本信息、绩效表现、项目经历、能力特长等分类
3. **信息呈现** - 以结构化方式向 HR 呈现分析结果
4. **信息确认** - HR 确认信息准确性，补充或修正

**呈现格式示例：**

```markdown

- 姓名：张 XXX
- 职位：后端开发工程师 P6
- 部门：技术部 - 基础架构组

- 年度评级：S（前 10%）

- 项目 B：主导 XX 系统重构

- 有晋升意向

### 阶段 2：生成个性化访谈思路（≤300 字）

**在生成完整提纲前，必须先生成并确认访谈思路。**

**访谈思路内容：**
1. **访谈重点** - 基于员工信息，说明本次访谈的核心关注点
2. **问题策略** - 说明问题设计的侧重点和方式
3. **注意事项** - 提醒需要特别注意的敏感点或风险

**格式要求：**
- 不超过 300 字
- 简洁清晰，重点突出
- 便于 HR 快速理解访谈策略

**示例：**

```
【访谈思路】

本次绩效访谈聚焦三大重点：1）肯定连续高绩效表现，强化成就感；2）了解高负荷下的工作状态，评估 burnout 风险；3）探索晋升诉求，明确发展路径。

问题策略：以开放式问题为主，先肯定成绩建立信任，再深入探讨工作状态和发展诉求。避免直接询问"是否考虑离职"，而是通过"工作节奏感受"间接了解。

注意事项：员工近期工作负荷大，避免过度施压；有晋升意向但本期名额有限，需谨慎回应；如表露离职倾向，立即启动挽留评估。
```

### 阶段 2.5：组织诊断访谈的特殊处理 ⭐

**如果是组织诊断访谈，需要额外执行以下步骤：**

#### 1. 方法论选择

**提供两种组织诊断方法论供 HR 选择：**

| 方法论 | 特点 | 适用场景 |
|-------|------|---------|
| **杨三角理论** ⭐ | 3 个支柱（思维/能力/治理），简洁聚焦，源自中国企业实践 | 快速诊断、HR 主导、互联网公司、需要明确行动计划 |
| **韦斯伯德六盒模型** | 6 个维度（目的/结构/关系/奖励/领导/支持机制），全面系统 | 全面诊断、外部顾问、传统企业、需要深度分析 |

**默认推荐：** 互联网公司优先使用**杨三角理论**（更贴近业务、更易落地）

#### 2. 维度选择（杨三角框架）

根据员工角色选择最相关的诊断维度：

| 员工角色 | 思维维度 | 能力维度 | 治理维度 |
|---------|---------|---------|---------|
| **高层管理者** | 战略认同、使命感 | 领导力、战略思维 | 决策机制、信息透明 |
| **中层管理者** | 目标对齐、责任感 | 管理能力、业务专业 | 授权机制、流程支持 |
| **一线员工** | 工作动机、归属感 | 岗位技能、学习能力 | 工作流程、工具支持 |
| **技术专家** | 技术热情、创新意愿 | 技术深度、知识更新 | 技术决策权、资源支持 |
| **新员工** | 文化融入、期望管理 | 岗位胜任、快速学习 | 入职流程、导师机制 |
| **老员工** | 持续激励、意义感 | 知识更新、经验传承 | 参与决策、知识管理 |

#### 3. 维度选择（韦斯伯德六盒框架）

如选择韦斯伯德六盒模型，根据员工角色选择最相关的 3-6 个维度：

| 员工角色 | 优先维度 | 说明 |
|---------|---------|------|
| **高层管理者** | 目的、领导、结构 | 战略方向、领导力、组织架构 |
| **中层管理者** | 结构、关系、支持机制 | 跨部门协作、团队关系、流程支持 |
| **一线员工** | 奖励、支持机制、关系 | 激励机制、工具支持、团队氛围 |
| **技术专家** | 支持机制、目的、奖励 | 技术工具、目标对齐、认可机制 |
| **新员工** | 结构、关系、支持机制 | 组织清晰、融入情况、培训支持 |
| **老员工** | 目的、奖励、领导 | 战略认同、长期激励、领导变革 |

#### 4. 系统性问题引导

**必须包含引导员工指出"系统性问题"而非"个人抱怨"的追问话术：**

```
【现象层】你遇到了什么具体问题？能举个例子吗？

【影响层】这个问题对你/团队的工作有什么影响？

【机制层】你觉得这个问题是个例，还是其他人也会遇到？

【系统层】如果是普遍问题，你觉得是什么机制/流程导致的？
         （杨三角归因：思维/能力/治理哪个环节的问题？）

【建议层】如果让你设计一个机制来避免这个问题，你会怎么做？
```

#### 5. 杨三角归因引导

当员工提出问题时，引导其归因到三大支柱：

```
【思维归因】
- 这个问题和大家的意愿/动机有关系吗？
- 是不是大家对目标的优先级理解不一致？

【能力归因】
- 这个问题需要什么样的能力才能解决？
- 团队是否具备这些能力？

【治理归因】
- 现有的流程/制度/授权是否支持解决这个问题？
- 是什么机制导致这个问题反复出现？
```

#### 6. 组织诊断访谈思路示例（杨三角版）

```
【组织诊断访谈思路】（杨三角框架）

本次组织诊断聚焦三大支柱（基于中层管理者角色）：

1. **员工思维** - 目标认同、工作动机、激励感受
   - 重点：了解员工是否"愿意"为组织目标投入

2. **员工能力** - 能力匹配、培养支持、知识沉淀
   - 重点：了解员工是否"会"完成工作任务

3. **员工治理** - 流程效率、授权机制、信息透明
   - 重点：了解员工是否"被允许"充分发挥能力

问题策略：采用"现象→影响→机制→系统→建议"五层追问法，每个问题引导员工归因到杨三角的某一支柱（思维/能力/治理），避免停留在个人抱怨层面。

注意事项：强调保密原则，鼓励真实表达；如员工提出具体问题，引导其思考系统性原因（是意愿问题？能力问题？还是机制问题？）；记录时按三大支柱分类，便于后续制定针对性改进计划。

参考文档：references/yang-triangle.md
```

#### 7. 组织诊断访谈思路示例（韦斯伯德版）

```
【组织诊断访谈思路】（韦斯伯德六盒框架）

本次组织诊断聚焦四个维度（基于中层管理者角色）：

1. **结构** - 职责分工和跨部门协作流程
2. **关系** - 团队内外协作和冲突处理
3. **支持机制** - 工具、信息、培训支持
4. **奖励** - 激励机制与目标对齐

问题策略：采用"现象→影响→机制→系统→建议"五层追问法，引导员工从个人体验上升到系统性思考。避免停留在个人抱怨层面，每个问题都要追问"如果是普遍问题，是什么机制导致的"。

参考文档：references/weisbord-six-box-model.md

## 📐 杨三角理论问题库

### 模型介绍

杨三角理论（Yang's Triangle）由杨国安教授提出，从三个支柱评估组织能力：

```
                    员工思维
                   (Mindset)
                      ▲
                     / \
                    /   \
                   /     \
                  /       \
                 /         \
                /           \
               /             \
              /               \
             /                 \
            /                   \
           /                     \
          /                       \
         /                         \
        /                           \
       /                             \
      /                               \
     /                                 \
    /                                   \
   /                                     \
  /                                       \
 /                                         \
员工能力 ◄─────────────────────────► 员工治理
(Employee                                   (Governance)
 Competence)
```

**三大支柱核心问题：**
- **员工思维**：员工是否**愿意**为组织目标投入？
- **员工能力**：员工是否**会**完成工作任务？
- **员工治理**：员工是否**被允许**充分发挥能力？

**维度选择逻辑：**

根据员工角色和层级，自动选择最相关的诊断维度：

|---------|---------|---------|---------|

| **中层管理者** | 目标对齐、责任感 | 管理能力、业务专业 | 授权机制、流程支持 |

### 支柱 1：员工思维（Mindset）- "愿不愿意"

**诊断重点：** 员工是否有正确的价值观、工作动机和投入度

```
主问题：你觉得团队/公司的目标有意义吗？你自己的工作目标是什么？

追问提示（引导系统性思考）：
- 目标是如何传达给你的？通过什么渠道和频率？
- 你日常工作与这些目标的关联是什么？能举例说明吗？
- 什么能让你在工作中保持动力和热情？
- 【思维归因】如果目标没有很好地落地，是大家不认同目标，还是激励不到位？

主问题：你觉得付出和回报匹配吗？什么样的奖励最能激励你？

追问提示（引导系统性思考）：
- 你如何定义"回报"？薪酬、认可、成长机会、工作意义？
- 过去一年中，哪次认可让你印象最深刻？
- 你觉得什么样的行为会被奖励？什么样的行为会被忽视？
- 【思维归因】如果觉得不匹配，是激励制度本身的问题，还是评估标准不清晰？

主问题：用三个词形容团队氛围，会是什么？为什么？

追问提示（引导系统性思考）：
- 团队中提出新想法通常会发生什么？
- 当你犯错时，团队的反应是什么？
- 你觉得团队鼓励什么样的行为？不鼓励什么样的行为？
- 【思维归因】如果氛围有问题，是文化导向问题，还是管理机制问题？
```

### 支柱 2：员工能力（Employee Competence）- "会不会"

**诊断重点：** 员工是否具备完成工作所需的知识、技能和能力

```
主问题：你觉得胜任当前岗位最关键的 3 项能力是什么？你哪些方面已经达标？

追问提示（引导系统性思考）：
- 工作中最常遇到的能力挑战是什么？
- 你觉得自己和下一个职级的能力差距在哪里？
- 当遇到新挑战时，你通常如何学习？
- 【能力归因】如果有能力缺口，是缺乏培训资源，还是缺乏实践机会？

主问题：过去一年你参加过哪些培训？哪些最有帮助？

追问提示（引导系统性思考）：
- 你希望公司提供什么样的发展支持？
- 团队中有导师或辅导机制吗？效果如何？
- 新人加入团队通常如何学习？多久能上手？
- 【能力归因】如果培训不够，是资源投入问题，还是缺乏能力模型和培训规划？

主问题：如果有人接替你的工作，需要多久能上手？团队有关键人风险吗？

追问提示（引导系统性思考）：
- 你的工作经验和方法有沉淀下来吗？（文档/流程/工具）
- 团队中谁最有潜力？为什么？
- 你对自己的晋升路径清晰吗？
- 【能力归因】如果人才梯队有风险，是缺乏继任计划，还是知识管理不到位？
```

主问题：你清楚自己的职责边界吗？有没有"三不管"或"多人管"的情况？

追问提示（引导系统性思考）：

## 📦 韦斯伯德六盒模型问题库

### 模型介绍

韦斯伯德六个盒子模型（Weisbord's Six-Box Model）是经典的组织诊断工具，从六个维度评估组织健康度：

```
┌─────────────────────────────────────────┐
│              外部环境                    │
├─────────────┬─────────────┬─────────────┤
│   目的      │   结构      │   关系      │
│  (Purpose)  │ (Structure) │(Relationships)│
├─────────────┼─────────────┼─────────────┤
│   奖励      │   领导      │  支持机制   │
│  (Rewards)  │ (Leadership)│(Mechanisms) │
├─────────────┴─────────────┴─────────────┤
│              内部协调                    │
└─────────────────────────────────────────┘
```

**维度选择逻辑：**

根据员工角色和层级，自动选择最相关的 3-6 个维度进行深度提问：

| 员工角色 | 优先维度 | 说明 |
|---------|---------|------|
| **高层管理者** | 目的、领导、结构 | 战略方向、领导力、组织架构 |
| **中层管理者** | 结构、关系、支持机制 | 跨部门协作、团队关系、流程支持 |
| **一线员工** | 奖励、支持机制、关系 | 激励机制、工具支持、团队氛围 |
| **技术专家** | 支持机制、目的、奖励 | 技术工具、目标对齐、认可机制 |
| **新员工** | 结构、关系、支持机制 | 组织清晰、融入情况、培训支持 |
| **老员工** | 目的、奖励、领导 | 战略认同、长期激励、领导变革 |

---

### 维度 1：目的（Purpose）

**诊断重点：** 组织目标是否清晰、被理解、被认同

```
主问题：你对团队/公司的目标清晰吗？觉得这些目标有意义吗？

追问提示（引导系统性思考）：
- 目标是如何传达给你的？通过什么渠道和频率？
- 你日常工作与这些目标的关联是什么？能举例说明吗？
- 目标在执行过程中是否发生过变化？变化的原因和沟通方式是什么？
- 【系统性追问】如果目标没有很好地落地，你觉得是目标本身的问题，还是传达/执行机制的问题？

主问题：你觉得团队的目标设定合理吗？为什么？

追问提示（引导系统性思考）：
- 目标设定时有你参与吗？如果没有，你觉得参与进来会有什么不同？
- 目标之间的优先级清晰吗？当多个目标冲突时，如何处理？
- 【系统性追问】如果目标不合理，是目标制定的流程问题，还是信息不对称的问题？
```

---

### 维度 2：结构（Structure）

**诊断重点：** 组织架构、分工、流程是否支持目标实现

```
主问题：你对团队的组织架构和分工清晰吗？有没有模糊或重叠的地方？

追问提示（引导系统性思考）：
- 你清楚自己的职责边界吗？有没有"三不管"或"多人管"的情况？
- 当遇到跨团队/跨部门的事情时，通常如何处理？有明确的流程吗？
- 【系统性追问】如果分工不清晰，是组织架构设计的问题，还是业务变化太快导致的？

主问题：你觉得现有的工作流程顺畅吗？有哪些卡点？

追问提示（引导系统性思考）：
- 哪些流程让你觉得效率低？具体是哪个环节？
- 这些流程最初是怎么设计的？现在是否还适用？
- 【系统性追问】如果流程不顺畅，是流程本身的问题，还是执行不到位，或者是缺乏监督机制？
```

---

### 维度 3：关系（Relationships）

**诊断重点：** 人际关系、团队协作、冲突处理

```
主问题：你和团队成员的合作怎么样？有没有什么挑战？

追问提示（引导系统性思考）：
- 团队内部的沟通方式是什么？正式会议多还是非正式沟通多？
- 当出现分歧或冲突时，通常如何解决？有固定的机制吗？
- 【系统性追问】如果协作有挑战，是人员匹配的问题，还是协作机制/流程的问题？

主问题：你觉得团队之间的协作顺畅吗？有什么阻碍吗？

追问提示（引导系统性思考）：
- 跨团队协作时，常见的障碍是什么？信息不对称？目标不一致？资源竞争？
- 现有的协作机制（如例会、项目群、对接人）有效吗？
- 【系统性追问】如果协作不顺畅，是团队文化的问题，还是缺乏有效的协作机制？
```

---

### 维度 4：奖励（Rewards）

**诊断重点：** 激励机制是否公平、有效、与目标对齐

```
主问题：你觉得付出和回报匹配吗？为什么？

追问提示（引导系统性思考）：
- 你如何定义"回报"？薪酬、认可、成长机会、工作意义？
- 你觉得什么样的行为会被奖励？什么样的行为会被忽视？
- 【系统性追问】如果觉得不匹配，是激励制度本身的问题，还是评估标准不清晰的问题？

主问题：团队/公司的激励机制能激励你更好地工作吗？

追问提示（引导系统性思考）：
- 现有的激励方式有哪些？（物质/精神/成长）哪种对你最有效？
- 激励和目标达成之间有明确的关联吗？
- 【系统性追问】如果激励不到位，是资源不足的问题，还是激励设计的问题（如标准模糊、延迟满足）？
```

---

### 维度 5：领导（Leadership）

**诊断重点：** 领导力、决策、方向指引

```
主问题：你和上级的沟通怎么样？他能给你提供什么支持？

追问提示（引导系统性思考）：
- 上级多久和你做一次 1 对 1 沟通？沟通的内容通常是什么？
- 当你遇到困难时，上级能提供什么帮助？（资源/方向/决策）
- 【系统性追问】如果沟通不够，是上级管理幅度太大，还是缺乏定期的沟通机制？

主问题：你觉得团队的决策方式怎么样？你参与决策的机会多吗？

追问提示（引导系统性思考）：
- 决策通常是自上而下还是自下而上？哪种方式更有效？
- 你有过提出建议被采纳的经历吗？流程是怎样的？
- 【系统性追问】如果决策不够透明，是领导风格问题，还是缺乏决策机制和反馈渠道？
```

---

### 维度 6：支持机制（Support Mechanisms）

**诊断重点：** 工具、流程、信息、培训等支持是否到位

```
主问题：你有完成工作所需的工具和资源吗？还缺什么？

追问提示（引导系统性思考）：
- 哪些工具/系统对你的工作帮助最大？哪些最难用？
- 当需要新工具/资源时，申请流程是怎样的？通常需要多久？
- 【系统性追问】如果工具不到位，是预算问题，还是缺乏需求收集和评估机制？

主问题：你觉得团队的信息透明吗？能及时获取需要的信息吗？

追问提示（引导系统性思考）：
- 重要信息通常通过什么渠道传达？（会议/邮件/IM/文档）
- 有没有遇到过信息滞后或缺失导致工作受影响的情况？
- 【系统性追问】如果信息不透明，是缺乏信息共享机制，还是文化上倾向于"需要知道"原则？

主问题：你获得的培训和支持能满足你的成长需求吗？

追问提示（引导系统性思考）：
- 过去一年参加过哪些培训？哪些对你有帮助？
- 当遇到新挑战时，通常如何学习？（自学/导师/培训）
- 【系统性追问】如果培训不够，是资源投入问题，还是缺乏能力模型和培训规划？
```

## ⚠️ 注意事项

### 敏感话题处理

| 敏感话题 | 处理建议 |
|---------|---------|
| 薪酬不满 | 先倾听，不承诺，记录后反馈 |
| 上级矛盾 | 保持中立，了解事实，不站队 |
| 离职倾向 | 探索真实原因，评估挽留可能 |
| 绩效争议 | 基于事实，提供申诉渠道 |
| 职场骚扰 | 严肃对待，按流程上报 |

### 风险预警

**高风险信号（需特别关注）：**
- 员工表达强烈不满
- 提及法律/合规问题
- 情绪明显异常
- 涉及多人/团队问题
- 可能引发劳动争议

**处理方式：**
1. 保持冷静，继续倾听
2. 不做承诺，不承认过错
3. 详细记录，及时上报
4. 必要时暂停访谈，寻求支持

### 文化适配

**互联网公司特点：**
- 语气可轻松但保持专业
- 鼓励开放表达
- 重视员工体验
- 关注成长与发展
- 扁平化沟通风格

  - 追问：你哪些方面已经达标？哪些还有差距？
  - 追问：工作中最常遇到的能力挑战是什么？
  - 【能力归因】如果有能力缺口，是缺乏培训资源，还是缺乏实践机会？

**人才培养**
- 主问题：过去一年你参加过哪些培训？哪些最有帮助？
  - 追问：你希望公司提供什么样的发展支持？
  - 追问：团队中有导师制吗？效果如何？

**人才梯队**
- 主问题：如果有人接替你的工作，需要多久能上手？
  - 追问：你的工作经验有沉淀下来吗？（文档/流程/工具）
  - 追问：团队有关键人风险吗？有继任计划吗？
  - 【能力归因】如果人才梯队有风险，是缺乏计划，还是知识管理不到位？

#### 支柱 3：员工治理 - "允不允许"（15 分钟）

**职责分工**
- 主问题：你清楚自己的职责边界吗？有没有"三不管"或"多人管"的情况？
  - 追问：跨团队协作时，通常如何处理？有明确流程吗？
  - 【治理归因】如果分工不清晰，是架构设计问题，还是业务变化太快？

**流程效率**
- 主问题：日常工作中哪些流程让你觉得效率低？能举个例子吗？
  - 追问：这些流程最初是怎么设计的？现在还适用吗？
  - 追问：如果让你 redesign 一个流程，你会改哪个？怎么改？

**授权机制**
- 主问题：你工作中多大比例的事情可以自主决策？
  - 追问：一个典型决策需要经过多少人审批？通常要多久？
  - 追问：有没有因为等待审批而耽误工作的经历？
  - 【治理归因】如果授权不够，是领导风格问题，还是缺乏决策机制？

**信息透明**
- 主问题：你觉得团队的信息透明吗？能及时获取需要的信息吗？
  - 追问：重要信息通过什么渠道传达？（会议/邮件/IM/文档）
  - 追问：有没有遇到过信息滞后导致工作受影响的情况？

### 三、系统性追问（5 分钟）
- 刚才提到的问题中，你觉得哪些是个人问题，哪些是机制问题？
- 如果是机制问题，属于思维/能力/治理哪个方面的原因？
- 如果让你设计一个机制来避免这些问题，你会怎么做？

### 四、员工提问环节（5 分钟）
- 你有什么想问我的吗？
- 对公司、团队、制度等方面有什么想了解的？

### 五、总结与后续行动（5 分钟）
- 总结今天讨论的关键要点（按思维/能力/治理分类）
- 确认哪些问题是个人层面，哪些是系统层面
- 约定后续反馈时间和方式

## 📝 访谈记录区

- 记录时区分"个人问题"和"系统问题"
- 如员工提出敏感问题（如离职倾向），单独记录并上报

### 示例 2：绩效访谈提纲

```markdown
# 员工访谈提纲 - 绩效面谈

## 📌 访谈基本信息
- 员工姓名：张 XXX
- 职位/职级：后端开发工程师 P6
- 所属部门：技术部 - 基础架构组
- 入职时间：2023 年 3 月
- 访谈类型：绩效访谈
- 预计时长：45-60 分钟
- 访谈人：HR 李 XXX
- 访谈日期：2025 年 Q1 绩效周期

## 🎯 访谈目的
1. 回顾 Q4 绩效目标达成情况
2. 识别工作中的挑战和改进空间
3. 讨论 2025 年个人发展计划
4. 了解资源和支持需求

## 🔍 背景信息
- Q4 绩效自评：超出预期
- 主管评价：整体表现优秀，技术能力强
- 特殊情况：参与了 2 个核心项目，工作负荷较大
- 关注点：工作生活平衡、职业发展路径

## 💬 访谈提纲

### 一、开场暖场（5 分钟）
- 最近工作节奏怎么样？还适应吗？
- 刚过去的这个季度，整体感觉如何？
- 今天想和你聊聊绩效和发展的话题，大概需要 45 分钟左右，过程中有任何想法都可以随时说~

### 二、核心问题

#### 主题 1：绩效回顾（15 分钟）
- 主问题：回顾 Q4 的几个目标，你觉得自己完成得怎么样？
  - 追问：哪个目标完成得最满意？为什么？
  - 追问：有没有哪个目标觉得比较吃力？
  - 追问：如果给自己打分，1-10 分你会打几分？

- 主问题：这个季度你最有成就感的工作是什么？
  - 追问：能具体讲讲你在这个项目中的贡献吗？
  - 追问：这个成果对团队/业务有什么价值？

#### 主题 2：挑战与改进（10 分钟）
- 主问题：工作中遇到的最大挑战是什么？
  - 追问：你是如何应对的？
  - 追问：如果再来一次，会有什么不同做法？

- 主问题：有什么地方觉得可以做得更好吗？
  - 追问：具体是哪些方面？
  - 追问：需要什么支持来改进？

#### 主题 3：职业发展（10 分钟）
- 主问题：你对自己的职业发展有什么规划？
  - 追问：短期（1 年）和长期（3 年）目标分别是什么？
  - 追问：觉得目前和下一个职级的差距在哪里？

- 主问题：你希望公司提供什么样的发展支持？
  - 追问：培训、项目机会、导师指导等，哪个对你最有帮助？

#### 主题 4：工作状态（5 分钟）
- 主问题：最近的工作节奏感觉怎么样？
  - 追问：工作和生活的平衡还好吗？
  - 追问：有什么我们可以帮你改善的吗？

### 三、员工提问环节（5 分钟）
- 你有什么想问我的吗？
- 对公司、团队、制度等方面有什么想了解的吗？

### 四、总结与后续行动（5 分钟）
- 总结今天讨论的关键要点
- 确认双方约定的后续行动
- 约定下次跟进时间

## 📝 访谈记录区
[此处预留记录空间]

## ⚠️ 注意事项
- 员工近期工作负荷较大，关注其工作状态
- 职业发展是关注重点，准备职级晋升标准信息
- 如有薪酬相关问题，记录后反馈，不当场承诺
```

## 📚 方法论索引

本技能基于多种专业方法论，针对不同访谈场景提供定制化框架：

### 🏛️ 组织诊断类

| 方法论 | 适用场景 | 参考文档 |
|-------|---------|---------|
| **杨三角理论** ⭐ | 组织健康度评估、快速诊断、HR 主导 | [yang-triangle.md](references/yang-triangle.md) |
| **用途** | 从员工思维、员工能力、员工治理三个支柱诊断组织能力 |
| **核心价值** | 简洁聚焦，源自中国企业实践，更易制定行动计划 |
| **推荐** | 互联网公司优先使用 |

| **韦斯伯德六个盒子模型** | 组织健康度评估、全面诊断、外部顾问 | [weisbord-six-box-model.md](references/weisbord-six-box-model.md) |
| **用途** | 从目的、结构、关系、奖励、领导、支持机制六个维度诊断组织问题 |
| **核心价值** | 引导员工指出"系统性问题"而非"个人抱怨" |

### 🚪 离职访谈类

| 方法论 | 适用场景 | 参考文档 |
|-------|---------|---------|
| **推拉力理论** | 离职原因分析、挽留评估 | [push-pull-theory.md](references/push-pull-theory.md) |
| **用途** | 从推力、拉力、留任力、移动障碍四个维度分析离职决策 |
| **核心价值** | 系统评估挽留可能性，收集改进建议 |

### 👥 招聘需求类

| 方法论 | 适用场景 | 参考文档 |
|-------|---------|---------|
| **冰山模型** | 岗位画像对齐、人才评估 | [iceberg-model.md](references/iceberg-model.md) |
| **用途** | 从知识、技能、社会角色、自我认知、特质、动机六个层次定义岗位 |
| **核心价值** | 区分"可见能力"与"隐藏素质"，避免只看表面 |

### 📈 员工发展类

| **核心价值** | 先环境后个人，避免"培训万能论" |

### 🎯 职业发展类
